Musical parameters

  • Rhythm (includes BPM, note duration)
  • Melody/harmony (encompasses pitch, scale, motif)
  • Timbre (synthesis, EQ, filters, ADSR)
  • Audio effects (such as reverb, distortion, ring modulation)
  • Texture (how many voices? how do they interact musically?)

Extramusical parameters

  • Device acceleration (x, y, z), and hence velocity and distance
  • Device orientation (alpha, beta, gamma)
  • Device geolocation (latitude, longitude)
  • Touch events and gestures
  • External parameters (e.g. time of day, weather season, proximity to other active devices, song database metadata, Foursquare check-in data, online friends list from Facebook graph API)

Why web-based?

  • Uses Web Audio API for sound generation, no need for external .mp3 files; < 50KB!
  • All real-time synthesis and processing done in-browser, no need to contact server after initial load of instrument settings
  • Easy to save, share, access

Community

  • Builders can share their instruments with others
  • Composers can write pieces for specific instruments and ensemble configurations
  • Performers can play with pre-made instruments, modify existing instruments, commission pieces for repertoire
  • ... and of course, all content providers can choose their pricing scheme
  • Users function as all of the above

Preset 2

  • simple sine wave + sawtooth wave + custom wave
  • alpha, beta, gamma <--> melodic motion along pentatonic scale for respective oscillators

Preset 3

  • simple sine wave + sawtooth wave + custom wave
  • algorithmically generated melodies based on the first three voices of Contrapunctus I from J.S. Bach's The Art of Fugue BWV 1080
  • alpha <--> tempo
